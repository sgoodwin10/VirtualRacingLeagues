//BUTTONS

Interactive button components with multiple variants, sizes, and states. Designed for clear visual hierarchy and accessible interactions.

Button Variants
Standard Variants
Primary for main actions, Secondary for alternatives, Ghost for subtle actions, Outline for emphasis without fill
Semantic Variants
Danger for destructive actions, Success for confirmations, Warning for caution states
Button Sizes
Size Scale
SM: 28px height, Default: 36px, LG: 44px, XL: 48px
Icon Buttons
Icon-Only Buttons
Square buttons with icons only. Available in all sizes and variants.
Button Groups
Grouped Actions
Related actions grouped together with connected borders
Icon Button Group
Navigation controls or toggle actions
Disabled State
Disabled Buttons
50% opacity with not-allowed cursor. All variants support disabled state.
USAGE GUIDELINES

Use .btn-primary for main call-to-action. Limit to one primary button per section. Use .btn-secondary for alternative actions. Use .btn-ghost for tertiary or repetitive actions like table row actions. Use .btn-danger only for destructive actions.
